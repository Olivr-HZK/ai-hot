# INS 热度分规则方案（超过 100 压缩）

## Summary
INS 热度分按“点赞 + 评论加权”与创作者近 7 日基线比较。基础分为 100 分制：低于或等于 100 时显示原始比例分；超过 100 的部分仿照 TikTok / X 的热度分做压缩，避免极端爆款把分数拉得过大。

## 规则

### 1. 原始互动热度
```text
rawHeat = likes + 8 * comments
```

### 2. 近 7 日创作者基线
```text
creatorAvgHeat7d = avg(likes + 8 * comments)
heatRatio = rawHeat / creatorAvgHeat7d
rawInsHeatScore = heatRatio * 100
```

如果该创作者近 7 日历史帖子数不足 3 条，则使用 27 个高质量博主整体近 7 日中位热度作为临时基线。

### 3. 超 100 压缩
```text
if rawInsHeatScore <= 100:
    insHeatScore = rawInsHeatScore
else:
    over = rawInsHeatScore - 100
    insHeatScore = 100 + 50 * over / (over + highScoreK)
```

当前：
```text
highScoreK = 600
```

因此分数不会简单显示几百、几千或几万，而是在超过 100 后逐渐靠近 150，用于表达“明显爆款”，同时保留排序差异。

### 4. 进入审核门槛
```text
insHeatScore >= 50
```

同时保留最低绝对线：
```text
likes >= 80
comments >= 2
```

手动 INS 搜索工作流可使用更高互动门槛，例如：
```text
likes >= 500
comments >= 10
```

## 分档
```text
爆款：>= 140 分
高热：100-139 分
可审：50-99 分
观察：30-49 分
过滤：< 30 分
```

热度分只决定是否进入审核，不替代产品手册、安全、视觉去重和 UA 审核。
