# X 搜索词扩展与 20 个上限控制计划

## Summary
- 当前 X 主配置搜索词是 **17 个**。
- X 搜索词可以扩展，但上限固定为 **20 个**，最多只新增 3 个。
- 当前 `.env` 为 `X_PAGES_PER_TERM=5`，因此扩展后理论最大 X 搜索分页请求为 **20 × 5 = 100 次**。

## Key Changes
- 新增配置：
  - `x_scrape.max_search_queries = 20`
- X scraper 执行前处理搜索词：
  - 去空、去重。
  - 保留原有高优先级 PDF workflow 词。
  - 超过 20 个时按优先级裁剪，只执行前 20 个。
  - 日志打印最终执行词数与理论最大分页请求数。
- 搜索词扩展方向：
  - 保留当前 17 个核心 workflow / prompt / photo-to-video 词。
  - 新增 3 个真人视觉素材方向词：
    - `AI fashion portrait prompt`
    - `AI editorial portrait prompt`
    - `cinematic portrait prompt`
  - 暂不加入过泛的 `fashion photoshoot`、`wedding photo`、`couple photoshoot`，避免 API 消耗和无关传统摄影内容上升。
- Stage0 反馈调节同步修改：
  - Stage0 可优化 `x_scrape.search_queries`，但数量不得超过 20。
  - Stage0 必须保留 PDF 样例相关核心词，如 `ChatGPT Seedance prompt workflow`、`GPT Images Seedance prompt`、`photo to video storyboard prompt`。
  - Stage0 新增词只能来自 AI workflow、prompt、照片转视频、真人视觉素材复用方向。
  - 负反馈命中的明星街拍、动漫、传统摄影业务词应进入 X 拒绝词，而不是替换成新的泛娱乐搜索词。

## Test Plan
- 当前配置读取后，确认 X 搜索词数量为 17。
- 扩展后，确认 X 搜索词数量为 20。
- 人为加入第 21 个词时，X scraper 实际只执行前 20 个并打印裁剪 warning。
- Stage0 反馈优化后，输出规则文件中的 `x_scrape.search_queries` 不超过 20。
- 理论最大请求数应显示为 `20 × X_PAGES_PER_TERM`，当前即 100。
