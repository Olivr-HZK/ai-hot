# 成品广告素材反向优化产品文档、筛选逻辑与提示词计划

## Summary
- 参考 `成品广告.png` 与 `成品广告.mp4`，把“已投放成功广告”的共性沉淀到产品手册、规则关键词和模型审核提示词中。
- 两个素材共同体现的高价值方向是：单张真人照片上传、强 before/after、角色/身份/梦幻场景转换、竖版移动端广告结构、明确 CTA、模板库卖点。
- 后续 TikTok / X / INS 筛选应更积极识别这类“可直接复用为广告成片或产品模板”的素材，并继续排除擦边、IP 依赖、低质 fantasy/anime 内容。

## 成品广告洞察
- `成品广告.png`：
  - 表现方式：左侧原始头像/肖像，箭头指向右侧直播间/主播身份成片，主标题 `STREAM DREAM`，副标题 `FROM PORTRAIT TO LIVE MOMENT`。
  - 可沉淀方向：真人肖像转直播/主播/creator persona，头像到生活化动态场景，photo-to-live-moment，强身份升级感。
  - 产品归因：优先 Kavi / Toki；若强调肖像质量和风格化，也可辅助 Evoke。
- `成品广告.mp4`：
  - 表现方式：竖版 9:16，开头产品结果吸睛，中段展示 `Upload a image`，再展示儿童照片变公主/精灵/童话角色，结尾 `Create Now`、`500+ Image & Video AI Template`、`Daily Updates on Viral Trends`、`Free Trial, Cancel Anytime`、App Store / Google Play。
  - 可沉淀方向：单张照片上传、儿童/家庭安全 fantasy portrait、dress-up / princess / fairy / storybook 模板、结果轮播、模板库和趋势更新卖点。
  - 产品归因：优先 Kavi / Toki；静态高质量肖像模板也可辅助 Evoke；不归 Avatar，除非出现头像拼图/社交小游戏机制。

## Key Changes
- 更新 `references/product_material_requirements.md`：
  - 新增“已验证广告素材模式”章节，明确这些方向为高优先级：single-photo upload、before/after、portrait-to-live-scene、streamer/creator persona、dream/fantasy dress-up、storybook/princess/fairy、template library、CTA/end-card。
  - 在 Kavi / Toki 中提高“照片到角色身份/梦幻场景/短视频广告成片”的权重；在 Evoke 中补充“高质量真人肖像风格化可作为素材库参考”。
  - 增加安全边界：儿童/家庭 fantasy portrait 可以接受，但必须 family-safe；禁止成人化、擦边、IP 公主/迪士尼等版权依赖、低质 AI fantasy/anime。
- 更新规则关键词与筛选逻辑：
  - TikTok/X/INS 产品匹配关键词加入：`single photo upload`、`upload an image`、`portrait to live moment`、`stream dream`、`streamer transformation`、`creator persona`、`dream portrait`、`storybook portrait`、`princess portrait`、`fairy portrait`、`dress up template`、`image/video template`、`template library`、`create now`。
  - X team demand 的 workflow/photo 判断中，把“广告成片结构、上传照片 UI、模板库卖点、CTA end-card”作为产品相关证据。
  - 对泛 fantasy/anime 不放宽：只有真人照片转换、明确产品玩法或广告成片结构时才加分。
- 更新模型审核提示词：
  - `ua_material_review`、`x_team_product_review`、`ins_product_review` 加入成品广告参考标准。
  - 模型需要优先通过“能直接复用为投放广告或模板库参考”的素材：有原图到结果、强视觉转变、明确角色身份、移动端 UI/CTA、模板库卖点。
  - 模型需要拒绝：纯漂亮 fantasy 图、IP/明星/动漫角色、儿童成人化、擦边、没有上传/模板/产品玩法证据的普通写真。
- 保持既有推送规则：
  - 产品范围仍只保留 Evoke / Toki / Kavi / Avatar。
  - 命中 Kavi/Toki/Evoke 且具备广告投放参考价值时可标记 `ALL`。
  - Avatar 仍只在头像生成 + 拼图 + Facebook 社交分享机制明确时命中。

## Test Plan
- 用 `成品广告.png` 的文本描述构造样例：应命中 Kavi/Toki，推送对象为 `ALL`。
- 用 `成品广告.mp4` 的文本描述构造样例：应命中 Kavi/Toki，儿童 fantasy portrait 因 family-safe 且有 upload/template/CTA 证据可通过。
- 普通 princess/fairy/anime 美图、无上传/模板/产品玩法证据：不得通过产品审核。
- 儿童 fantasy 但成人化、擦边、IP 依赖或低质：必须被安全/产品审核拒绝。
- 直播间/streamer 方向若只是普通主播照片、无 AI 转换或模板证据：不得通过。
- 回归验证：Evoke / Toki / Kavi / Avatar 原有关键词仍可命中；Deep Think / Zensi 不回流；飞书字段不变。

## Assumptions
- 这两个成品广告代表“已验证可投放素材形态”，优先用于提高筛选和审核权重，不改变平台抓取方式。
- `成品广告.mp4` 中儿童素材只作为 family-safe 照片模板参考，不鼓励任何成人化、擦边或敏感表达。
- 本次优化不新增产品，不改变每日推送上限，只调整产品文档、关键词规则、筛选判断和模型审核提示词。
