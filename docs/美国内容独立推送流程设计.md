# 美国内容独立推送流程设计

## Summary
- 新增一条隔离的“美国英语内容推送”流程，默认不接入日常任务、不写入多维表格，只通过飞书 webhook 推送。
- 支持 TikTok / X 两个平台，搜索词自动读取当前 `references/tiktok_feedback_optimization_rules.json` 中最新版 TikTok 与 X 搜索词。
- v1 采用“双模式”：默认从本地缓存/已有抓取结果 dry-run 调试；显式 `--scrape` 时才重新抓取，避免误耗 API。
- 当前只实现为人工/独立入口，热度门槛按 UA 定向放宽版设计但默认不启用硬过滤。

## Key Changes
- 新增独立入口，例如 `scripts/us_t1/us_content_push.py`，输出只落到 `skill_runs/us_t1/`，不覆盖主流程的 `hotspots.json`、平台 `hotspots_*.json`、scraper `filtered-result.json`。
- 数据获取：
  - TikTok：读取当前 `scrape.search_queries`，显式抓取时使用美国代理/地区参数，并限定英语；缓存模式读取最近 TikTok raw/checkpoint。
  - X：读取当前 `x_scrape.search_queries`，显式抓取时为每个 query 增加美国与英语限定，例如 `lang:en` + 美国语义/地点约束；缓存模式读取最近 X raw/checkpoint。
  - 所有候选打标 `usT1Targeting`，记录 `platform`、`sourceQuery`、`country=US`、`language=en`、`sourceMode=cache|scrape`。
- 筛选链路：
  - 先做美国/英语确认：TikTok 优先用 `locationCreated=US` 或文本/作者/标签美国信号；X 优先用 `raw_source.lang=en`，并要求文本/作者/地点/搜索条件命中美国信号。
  - 不启用素材数量上限，不参与现有 `ua_geo_daily_max`、`push_caps`。
  - 热度门槛配置为 UA geo 放宽版：`min_play_count * 0.5`、`min_comment_count * 0.5`，但 v1 默认 `enforce_heat=false`，只记录 `heatPass`。
  - 必须不命中现有排除词。
  - 必须通过产品手册审核，产品只允许 Evoke / Toki / Kavi / Avatar。
  - 必须再通过第二道 OpenRouter 美国投放审核，判断是否适合美国英语投放、是否安全、是否有广告素材/模板复用价值。
- 推送：
  - 所有通过项标记 `pushObject="ALL"`，因为目标是美国投放素材且必须通过产品手册。
  - 生成独立 JSON：`skill_runs/us_t1/latest.json` 与 `skill_runs/us_t1/runs/<runId>.json`。
  - 只调用飞书 webhook 推送卡片，不调用 bitable 写入逻辑。
  - 飞书卡片标题使用“美国内容候选推送”，包含平台、热度、链接、推荐产品、两道审核理由。

## Interfaces
- 新脚本参数：
  - `--platforms tiktok,x`：默认两个平台都跑。
  - `--source cache|scrape`：默认 `cache`；只有 `scrape` 才请求 API。
  - `--enforce-heat`：默认关闭；打开后才按 UA 放宽热度门槛过滤。
  - `--dry-run`：只打印和写 JSON，不发飞书。
- 新配置可放入 `.env.example`：
  - `US_T1_PUSH_ENABLED=false`
  - `US_T1_DEFAULT_SOURCE=cache`
  - `US_T1_COUNTRY=US`
  - `US_T1_LANGUAGE=en`
  - `US_T1_ENFORCE_HEAT=false`
  - `US_T1_REVIEW_MODEL=qwen/qwen3.7-max`
- 不修改现有日常流程入口，不修改飞书表格字段，不新增定时任务。

## Test Plan
- 缓存模式：用现有 TikTok/X raw 数据运行，确认不会触发 API、不会写主流程文件、不会写飞书表格。
- 搜索词同步：修改规则文件中的 TikTok/X 搜索词后，独立流程自动读取最新列表。
- 地区语言过滤：非英语、非美国信号内容被拒绝；英语且美国信号内容进入审核。
- 热度门槛：默认只记录 `heatPass` 不过滤；加 `--enforce-heat` 后才过滤低热内容。
- 审核链路：命中排除词、产品手册拒绝、美国投放审核拒绝的内容均不推送。
- 推送链路：通过项只发飞书 webhook，确认 bitable create/update 不被调用。
- 回归：现有 `run_pipeline.py`、TikTok/X/INS 日常流程不引用 `scripts/us_t1/`。

## Assumptions
- “先不要生效”表示不接入 Windows 定时任务、不接入 `run_pipeline.py`，只提供人工独立入口。
- 美国限定在 TikTok 上优先使用 `locationCreated=US`，在 X 上使用 `lang:en` 与美国文本/地点信号组合判断；若平台 API 缺少可靠地理字段，不用猜测放行。
- “通过 OpenRouter 的 AI 审核”理解为产品手册审核之后的第二道独立模型审核。
