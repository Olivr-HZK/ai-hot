# 飞书反馈字段统一与 ALL 推送对象改造计划

## Summary
- 后续所有通过筛选、审核确认推送的素材，`推送对象` 统一写为 `ALL`，不再区分 `UA` / `产品`。
- 飞书反馈改为单一新字段：`采纳意愿`，使用评分字段：`1星=否决`、`2星=中`、`3星=高`。
- 反馈原因统一读取新字段：`原因`。
- 旧字段不删除，但程序后续不再读取旧字段：`UA采纳意愿`、`产品采纳意愿`、`浩鹏意愿`、`UA原因`、`产品原因`、`浩鹏原因`。

## Key Changes
- 推送对象统一：
  - 在最终写入飞书前强制所有通过项 `pushObject="ALL"`。
  - 飞书写表 payload 中 `推送对象` 固定写 `ALL`。
  - 反馈硬过滤不再做 UA / 产品单侧保留或重定向，只判断“保留 / 拒绝”。

- 推送上限：
  - 停用 UA 侧、产品侧、UA 国家定向、产品视频等分侧上限。
  - 只保留每日总推送上限：`15` 条。
  - 超过 15 条时按最终热度/排序保留前 15 条。

- 飞书反馈读取：
  - 新增统一反馈解析：`采纳意愿 -> material_acceptance`。
  - 解析规则：
    - `1` / `1星` -> `否决`
    - `2` / `2星` -> `中`
    - `3` / `3星` -> `高`
  - `原因` 作为唯一反馈原因字段。
  - 旧反馈字段全部设为忽略/只读，不参与 Stage0、硬过滤、视觉去重、统计分析。

- Stage0 与历史反馈逻辑：
  - `3星/高`：高质量正反馈，用于提升相关关键词、创作者、素材方向权重。
  - `2星/中`：可用素材，作为轻正向反馈。
  - `1星/否决`：负反馈，用于降权、拒绝词学习和历史硬过滤。
  - 所有平台统一按素材级反馈处理，不再分别计算 UA 高质量率、产品高质量率。

- 飞书写入保护：
  - 程序不写入 `采纳意愿` 和 `原因`，这两个字段由人工反馈填写。
  - 继续不读取、不写入 `素材类型`、`prompt反推结果`。
  - `自动prompt获取`、平台、简介、链接、热度评分等现有写入字段保持不变。

## Implementation Targets
- 更新飞书字段工具与读取逻辑，统一使用 `采纳意愿` / `原因`。
- 更新 `feishu_push` 写表逻辑，固定 `推送对象=ALL`。
- 更新 Stage0 反馈调节、历史反馈硬过滤、视觉去重、统计脚本、手动 INS 工作流等所有读取飞书反馈的模块。
- 更新配置/文档中的推送规则说明：后续素材只分“通过 / 不通过”，不再分 UA / 产品推送对象。

## Test Plan
- 构造飞书反馈行：
  - `采纳意愿=1星` -> 识别为 `否决`。
  - `采纳意愿=2星` -> 识别为 `中`。
  - `采纳意愿=3星` -> 识别为 `高`。
- 旧字段有值、新字段为空：程序应视为未标记，不读取旧字段。
- 任意平台通过筛选的素材写表时，`推送对象` 都应为 `ALL`。
- 构造 20 条通过素材，最终只保留 15 条。
- 验证飞书写表 payload 不包含旧反馈字段，也不写入 `采纳意愿` / `原因`。
- 运行 Python 语法检查与现有推送上限回归测试。

## Assumptions
- 新飞书字段名固定为：`采纳意愿`、`原因`。
- `采纳意愿` 是飞书评分字段；程序也兼容读取文本形式 `1星/2星/3星`。
- 历史旧字段不迁移、不兜底；从本次改造后只以新字段作为反馈来源。
