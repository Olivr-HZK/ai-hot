# 飞书新增 UA 反馈字段接入计划修订版

## Summary
- Stage0 将 `UA采纳意愿` 和 `浩鹏意愿` 都作为 UA 反馈读取。
- 后续处理中 `低` 与 `否决` 等价，统一视为 UA 负反馈。
- `素材类型`、`prompt反推结果` 与本项目无关：不读取、不写入、不参与分析。

## Key Changes
- 扩展 UA 反馈读取：
  - UA 意愿字段：`UA采纳意愿`、`浩鹏意愿`。
  - UA 原因字段：`UA原因`、`浩鹏原因`。
  - 产品反馈字段保持不变。
- UA 多来源合并规则：
  - 任一来源为 `高`，该条按 UA 高质量反馈计入。
  - `低` = `否决`，统一按 UA 负反馈处理。
  - 若为 `中` + `低/否决` 冲突，Stage0 按 1:1 权重计入 UA 中和 UA 负反馈。
  - 单字段存在时沿用该字段结果。
- 同步使用范围：
  - Stage0 反馈调节读取并分析两个 UA 来源。
  - 历史反馈硬过滤同步识别 `浩鹏意愿/浩鹏原因`，并将 `低/否决` 都视为负反馈。
  - 视觉去重如读取历史 UA 反馈，也改为使用合并后的 UA 反馈。
- 写入保护：
  - 飞书写入逻辑不新增字段。
  - 明确将 `浩鹏意愿`、`浩鹏原因`、`素材类型`、`prompt反推结果` 设为只读/忽略字段。
  - `素材类型`、`prompt反推结果` 不进入任何读取列表。

## Test Plan
- 只有 `浩鹏意愿=高`：Stage0 识别为 UA 高质量反馈。
- 只有 `浩鹏意愿=低`：Stage0 识别为 UA 负反馈，与 `否决` 等价。
- `UA采纳意愿=中` + `浩鹏意愿=低/否决`：Stage0 按 1:1 计入 UA 中和 UA 负反馈。
- `UA采纳意愿=低` + `浩鹏意愿=高`：按 UA 高质量反馈计入。
- 带有 `素材类型`、`prompt反推结果`：确认不会被读取或写入。
- 验证飞书写入字段不包含四个新增字段。
- 运行反馈解析相关单测/脚本检查与 Python 语法检查。

## Assumptions
- 现有 UA 字段标准名为 `UA采纳意愿`。
- `浩鹏意愿` 与 `UA采纳意愿` 权重同属 UA 反馈类。
- 冲突反馈的 1:1 权重用于 Stage0 反馈调节；硬过滤阶段仍将明确 `低/否决` 作为负反馈信号处理。
